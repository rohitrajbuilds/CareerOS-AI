# CareerOS AI Permissions Explanation

## Requested Chrome permissions

- `storage`: settings, encrypted local profile data, and application state
- `tabs`: identify and coordinate supported pages
- `alarms`: background lifecycle-safe refresh and maintenance
- `sidePanel`: main operator workspace
- `scripting`: content script injection for supported ATS flows
- `activeTab`: user-initiated access to the current job application page

## Host access

- Workday
- MyWorkdayJobs
- LinkedIn
- Greenhouse
- Lever
- Localhost for development

These permissions support form detection, field analysis, and user-triggered autofill on supported job application experiences.
