CareerOS AI

Permissions Explanation

This page explains why the extension requests each Chrome permission and supported host access pattern.

Chrome API permissions

  • storage: saves user settings, profile state, and encrypted local records.
  • tabs: identifies the active tab and supported application pages.
  • alarms: schedules background refresh and lifecycle-safe maintenance tasks.
  • sidePanel: opens the main workspace experience inside Chrome.
  • scripting: injects content scripts into supported job application sites.
  • activeTab: performs user-initiated actions against the current tab without broader permanent access.

Host permissions

CareerOS AI only requests host access for supported job application providers and local development APIs:

  • Workday and MyWorkdayJobs
  • LinkedIn
  • Greenhouse
  • Lever
  • Localhost backend during development

Host access is used to detect job forms, read field context, inject content scripts, and fill fields after the user activates the extension workflow.
